# Security

## Signing

- CI signs the artifact, SBOM, attestation, and manifest.
- A local RSA keypair is generated by `ci/scripts/sign_artifact.sh`.
- The private key is not meant to be committed.
- The public key is copied to the gateway and client trust stores.

## Verification

- The gateway verifies files before caching them.
- The client verifies files again before install.
- Verification works offline with local public keys.
- The client rejects downgrade or replay attempts.

## Threat Model

This prototype is designed to detect:

- tampered artifacts or metadata
- modified files served by a gateway
- replay or downgrade attempts
- temporary loss of connectivity between components

This prototype does not fully address:

- compromise of the CI signing environment
- host-level compromise of the demo machine
- transport-layer security between services

## Key Rotation

1. Generate a new signing keypair.
2. Distribute the new public key.
3. Trust both keys during migration.
4. Start signing with the new private key.
5. Remove the old key after migration.
